### US-001: Define situation and role domain model
Status: in_progress
Depends on: None

Description:
As a developer, I want a clear in-code model for situations and roles so round generation can enforce player-count rules reliably.

Acceptance Criteria:
- [ ] Create structured situation objects with id, name, guesser prompt, clue checklist, and roles with mandatory flags.
- [ ] Role schema includes communicationMode, interactionWith, and clueSignals.
- [ ] Example: with 5 players and a situation containing 3 mandatory + 2 optional roles, the situation is valid because 4 performers can be assigned.
- [ ] Negative case: with 4 players and a situation requiring 4 mandatory performer roles, the situation is rejected because only 3 performers are available.
- [ ] Show a deterministic error message when no situation is compatible.
